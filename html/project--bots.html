Project

Bots

Challenges

  • As a bot author,
  • I would like to learn how to build my bots quick and easy,
  • so that I can use them to reduce calls, improve the bots performance and share them with other colleagues.

Challenges1/3

  • The current design was mostly inspired by a competitor product

Challenges2/3

  • Only high skilled developers were able to understand how to use it to build a bot

Challenges3/3

  • No content, idea or strategy to onboard new users

Ideation

  • Feedback from Sales: Onboarding was the biggest blocker for customers
  • Pick this most important topic "Onboarding"
  • Advocate for UX to inspire the right people to support new design thinking methods
  • Start a new ideation process
  • Organise the right target group for user testing

Ideation1/3

  • Introduce remote mockup sessions
  • Inspire people with multiple backgrounds to join
  • Crazy 8 was a bit to much for the start
  • Communicate early what the remote participants need for the session
  • 3 mockups in fifteen minutes
  • Each participant gets 5 minutes to explain their ideas

Ideation2/3

  • Vote remote for the best idea
  • Use Confluence as a voting platform
  • Every participant has three stars they can share
  • Give a deadline for the votes
  • Celebrate the most voted idea

Ideation3/3

  • Create a prototype for testing
  • Discuss the prototype with management and development
  • Do remote whiteboard sessions with them
  • Prepare an MVP for user testing

Prototyping

  • Stakeholders asked for a vision that looks like a modern application
  • Extend the design language available at Genesys
  • Introduce a guided prototyping approach
  • Give the whole thing a catchy name ;)

Prototyping1/3

  • Reduce to the right minimum elements
  • Create an image mark for the product
  • Inspire the technical copywriter do do something new
  • Introduce the new helper bot and let it guide through a whole onboarding user journey

Prototyping2/3

  • Introduce UI best practices
  • Use micro animations to support the Information Architecture
  • Create a modern icon set
  • Use slides instead of modals to avoid caging users

Prototyping3/3

  • Cover three most important user journeys
  • Give a guide for remote testing
  • Click here to test the prototype

Survey

  • Collaborate with product management to line up experienced bot authors for user testing
  • Introduce a new test template in Confluence for a higher evaluation performance
  • Use the capabilities of Confluence to give maximum visibility to the data
  • Communicate the findings to the team and all stakeholders
  • Integrate the findings into the planning of the next iteration of the product
